# SIGNAFYai BETA BUILD INSTRUCTIONS
## Objective
Build a professional beta SaaS platform called **SignafyAI**.  
SignalForge is a multi-tenant AI growth operating system connected to **n8n** for automation.  
The system must help businesses across different industries generate leads, create SEO-oriented social content, manage social responses in brand voice, and drive traffic back to their websites.

This is a **beta launch build** intended to be:
- developed in Claude
- committed to GitHub
- deployed on Vercel
- connected to self-hosted or cloud n8n as the automation engine

---

# CORE PRODUCT VISION

SignafyAI should feel like a premium SaaS product for agencies, consultants, and brands that want semi-autonomous growth operations.

The platform should do 4 major things:

1. **Lead Generation**
   - Generate business leads across industries and local markets
   - Pull, organize, and score prospect data
   - Allow filtering by niche, city, service type, company size, keyword intent, and platform/source
   - Support lead enrichment and lead scoring
   - Store leads inside the SaaS for review/export/action

2. **SEO and Content Automation**
   - Generate SEO-based captions and content for social media posts
   - Highlight keyword intent naturally in captions, not in a spammy way
   - Support image post, carousel post, short video post, and blog/social repurposing
   - Produce platform variations for Instagram, TikTok, YouTube Shorts, Facebook, X, and LinkedIn
   - Create content tied to search intent, local relevance, and offer relevance

3. **Traffic Generation**
   - Create workflows that push traffic back to websites through strategic content, CTA sequencing, lead magnets, profile optimization suggestions, and post scheduling
   - Support internal link suggestions for websites and landing pages
   - Support campaign logic around local SEO, offer campaigns, educational content, testimonial content, and retargeting hooks

4. **Social Response Automation**
   - Answer inbound questions and comments on Instagram, YouTube, and TikTok
   - Responses must sound natural and brand-matched
   - Allow different response modes:
     - fully autonomous
     - suggested reply for approval
     - high-risk questions escalated to manual review
   - Must avoid robotic tone, repetition, spammy CTA behavior, or policy-unsafe actions

---

# IMPORTANT PRODUCT STRATEGY

Do NOT build this like one giant AI bot.

Build it as:
- a **control dashboard**
- connected to **specialized workflow engines in n8n**
- with modular services for:
  - lead generation
  - enrichment
  - content generation
  - posting orchestration
  - social response handling
  - analytics
  - tenant settings

Each of these should be isolated as its own workflow/service area so the platform is stable, scalable, and easier to debug.

# PHASE 1: N8N FIRST

## Goal
Build the automation foundation first in n8n before building the full SaaS interface.

## N8N Architecture Requirements
Create separate n8n workflow groups for the following systems:

### 1. Lead Discovery Engine
Should:
- accept user inputs from webhook or API
- inputs include:
  - industry
  - location
  - target keywords
  - business type
  - website URL
  - ideal customer type
  - desired platform/source
- find leads from supported sources and normalize the data
- enrich with:
  - company name
  - website
  - email if available
  - social links if available
  - phone if available
  - niche tags
  - local relevance
  - lead score
- send normalized results to database/API

### 2. Content Intelligence Engine
Should:
- accept business profile data and campaign goals
- generate:
  - caption ideas
  - short-form video prompts
  - post descriptions
  - keyword clusters
  - hashtag ideas
  - SEO-based topic ideas
  - CTA recommendations
- support platform-specific formatting for:
  - Instagram
  - TikTok
  - YouTube
  - LinkedIn
  - Facebook
  - X
- output content to database with statuses:
  - draft
  - queued
  - approved
  - posted
  - needs review

### 3. Social Listening and Response Engine
Should:
- receive inbound comments, questions, and DM-like message events through available integrations/webhooks
- classify message intent:
  - question
  - objection
  - praise
  - complaint
  - buying signal
  - spam
- generate a natural response based on:
  - brand voice
  - tone settings
  - platform style
  - CTA rules
  - escalation thresholds
- output either:
  - auto-send response
  - draft response for user approval
  - escalate to human review

### 4. SEO Traffic Support Engine
Should:
- generate keyword clusters based on business niche
- identify search intent categories:
  - informational
  - local
  - commercial
  - transactional
- create content suggestions aligned to intent
- suggest landing page themes
- suggest internal linking ideas
- suggest FAQ ideas for websites
- suggest metadata ideas and headings for SEO pages
- send all outputs back to SaaS dashboard for user review

### 5. Campaign Scheduler Engine
Should:
- queue posts
- stagger timing by platform
- avoid obvious spam patterns
- support campaign calendars
- trigger content publish workflows
- update post status after publish attempt
- notify dashboard of success/failure

### 6. Analytics Sync Engine
Should:
- receive and store:
  - lead counts
  - content counts
  - reply counts
  - post engagement
  - clicks
  - site traffic snapshots
  - conversion events if connected
- create summaries for dashboard cards and reports

---

# N8N ENGINEERING RULES

1. Build all workflows to be modular and reusable
2. Use webhook entry points where possible so the SaaS can trigger them cleanly
3. Return structured JSON responses from each workflow
4. Use retries, fallback paths, and error logging
5. Every workflow should write logs to a database table or logging endpoint
6. Protect secrets and credentials carefully
7. Use queue-friendly design patterns so future scale is possible
8. Use environment variables for keys, URLs, tokens, and secrets
9. Add clear workflow naming and documentation for each workflow
10. Each workflow should be easy to test independently

# AUTOMATION SAFETY RULES

Add controls so the user can choose:
- fully autonomous
- approval required
- mixed mode

Mixed mode rules:
- low-risk content can auto-post
- questions with pricing/legal/medical/political sensitivity require review
- complaint handling should usually require approval
- high-buying-intent responses can be suggested with alerting

The system must never feel reckless or spammy.
